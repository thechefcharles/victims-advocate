--- DECISIONS ---

Decision 1 — Safety preferences naming (resolves Follow-up Q1)
ADOPT OPTION B. The `user_safety_settings` table stays with its current name. No `ALTER TABLE RENAME` migration. The code model in Domain 3.1 is named `SafetyPreference` and lives in `lib/server/applicant/safetyPreferenceService.ts`. The existing `lib/server/safety/settings.ts` becomes a thin re-export alias pointing to the new service. The underlying table name referenced in all queries remains `user_safety_settings`. Rationale: zero migration cost, zero risk to a working production table, precedent set by Domain 2.3.

Decision 2 — applicant_profiles table design (resolves Follow-up Q2 and Q6)
CREATE a new `applicant_profiles` table as a one-to-one extension of `profiles` (FK: `applicant_profiles.user_id UNIQUE → auth.users(id)`). Cardinality: one-to-one. Provider/agency/admin `profiles` rows have no corresponding `applicant_profiles` row. Dual-write: when Domain 3.1's `updateApplicantProfile` writes to `applicant_profiles`, it also writes the overlapping fields back to `profiles.personal_info` jsonb. The jsonb column is NOT dropped in this domain. The `applicant_profiles` table holds all fields from the `personal_info` zod schema as proper columns PLUS `profile_completion_pct integer` computed on write.

Decision 3 — trusted_helper_access schema (resolves Follow-up Q3)
CREATE `trusted_helper_access` with these columns:
- `id uuid PK DEFAULT gen_random_uuid()`
- `applicant_user_id uuid NOT NULL REFERENCES auth.users(id) ON DELETE CASCADE`
- `helper_user_id uuid NOT NULL REFERENCES auth.users(id) ON DELETE CASCADE`
- `granted_scope text[] NOT NULL DEFAULT '{}'` — array of action strings, not jsonb. Allowed values: `'intake:view'`, `'intake:edit'`, `'documents:upload'`, `'messages:read'`, `'profile:view'`. Stored as a Postgres text array. Validation enforced in service layer.
- `status text NOT NULL DEFAULT 'pending' CHECK (status IN ('pending','active','revoked'))`
- `granted_at timestamptz DEFAULT now()`
- `accepted_at timestamptz`
- `revoked_at timestamptz`
- `granted_by_user_id uuid NOT NULL REFERENCES auth.users(id)`
- `notes text`
- `created_at timestamptz DEFAULT now()`
- `updated_at timestamptz DEFAULT now()`
- UNIQUE `(applicant_user_id, helper_user_id)` — one grant record per pair; status tracks lifecycle.

Rationale for text array over jsonb: simpler to query (`'intake:edit' = ANY(granted_scope)`), no JSON parsing in policy evaluator, easier to index.

Decision 4 — applicant_bookmarks target type (resolves Follow-up Q4)
CREATE `applicant_bookmarks` with:
- `id uuid PK DEFAULT gen_random_uuid()`
- `applicant_user_id uuid NOT NULL REFERENCES auth.users(id) ON DELETE CASCADE`
- `target_type text NOT NULL CHECK (target_type IN ('provider','program','resource'))`
- `target_id uuid NOT NULL` — polymorphic UUID; no FK constraint (targets are in different tables)
- `position integer NOT NULL DEFAULT 0`
- `notes text`
- `created_at timestamptz DEFAULT now()`
- UNIQUE `(applicant_user_id, target_type, target_id)`

Rationale: Three target types confirmed. No per-type FK constraints to avoid cross-domain coupling. Uniqueness prevents duplicate bookmarks.

Decision 5 — applicant_preferences field set (resolves Follow-up Q5)
CREATE `applicant_preferences` with:
- `id uuid PK DEFAULT gen_random_uuid()`
- `user_id uuid NOT NULL UNIQUE REFERENCES auth.users(id) ON DELETE CASCADE`
- `accessibility_mode text NOT NULL DEFAULT 'none' CHECK (accessibility_mode IN ('high_contrast','large_text','screen_reader','none'))`
- `notification_channel_preference text NOT NULL DEFAULT 'in_app' CHECK (notification_channel_preference IN ('email','sms','in_app'))`
- `discovery_search_radius_miles integer NOT NULL DEFAULT 25`
- `discovery_default_state_code text`
- `intake_save_frequency_seconds integer NOT NULL DEFAULT 30`
- `created_at timestamptz DEFAULT now()`
- `updated_at timestamptz DEFAULT now()`

Does NOT include: locale (Domain 2.4), safety mode (user_safety_settings), language for translation (locale_preferences). Rationale: clean separation of concerns per analysis recommendation.

Decision 6 — resolveApplicantByUserId signature (resolves Follow-up Q7)
```typescript
resolveApplicantByUserId(
  userId: string,
  supabase: SupabaseClient
): Promise<ApplicantProfileRecord | null>
```
Pure read. No policy gate. Caller is responsible for authorization. Returns `null` (not throws) when no applicant_profiles row exists. Other domains (intake, case, support_request) consume this without needing to know about the policy layer.

Decision 7 — assertApplicantSelfOrGrantedHelper / helper acting-as semantics (resolves Follow-up Q8)
Helpers operate with a synthetic context: `{ userId: applicant_user_id, actingAs: helper_user_id, grantedScope: string[] }`. Routes that accept helper actions receive a standard `actor` object with `userId` set to the applicant they are acting for. The `actingAs` field identifies who is physically authenticated. Audit logs record BOTH. The policy evaluator for `trusted_helper:act_as` checks that (a) the authenticated session belongs to `helper_user_id`, (b) a grant record exists with `status='active'`, and (c) the requested action is in `granted_scope`.

Decision 8 — /victim/* URL paths (resolves Follow-up Q9)
URLs stay unchanged. `/victim/*` routes are not moved or redirected. Internal service/type naming uses `applicant`. This is a code-layer rename only.

Decision 9 — Trust signals (resolves Follow-up Q10)

```sql
-- Domain 3.1 — Applicant Domain
-- Creates applicant_profiles as a one-to-one extension of profiles
-- personal_info jsonb on profiles is NOT dropped here (dual-write transition)

CREATE TABLE IF NOT EXISTS public.applicant_profiles (
  id                          uuid PRIMARY KEY DEFAULT gen_random_uuid(),
  user_id                     uuid NOT NULL UNIQUE REFERENCES auth.users(id) ON DELETE CASCADE,

  -- Identity (mirrors personal_info jsonb fields as relational columns)
  preferred_name              text,
  legal_first_name            text,
  legal_last_name             text,
  pronouns                    text,
  gender_identity             text,
  date_of_birth               date,
  ethnicity                   text,
  race                        text,

  -- Contact
  street_address              text,
  apt                         text,
  city                        text,
  state                       text,
  zip                         text,
  cell_phone                  text,
  alternate_phone             text,

  -- Background
  occupation                  text,
  education_level             text,

  -- Communication preferences (kept here as profile-level, distinct from locale_preferences)
  interpreter_needed          boolean DEFAULT false,
  preferred_contact_method    text,

  -- Computed
  profile_completion_pct      integer NOT NULL DEFAULT 0 CHECK (profile_completion_pct BETWEEN 0 AND 100),

  created_at                  timestamptz NOT NULL DEFAULT now(),
  updated_at                  timestamptz NOT NULL DEFAULT now()
);

-- RLS
ALTER TABLE public.applicant_profiles ENABLE ROW LEVEL SECURITY;

-- Applicant: full access to own row
CREATE POLICY "applicant_profiles_self_select"
  ON public.applicant_profiles FOR SELECT
  USING (auth.uid() = user_id);

CREATE POLICY "applicant_profiles_self_insert"
  ON public.applicant_profiles FOR INSERT
  WITH CHECK (auth.uid() = user_id);

CREATE POLICY "applicant_profiles_self_update"
  ON public.applicant_profiles FOR UPDATE
  USING (auth.uid() = user_id)
  WITH CHECK (auth.uid() = user_id);

-- Platform admin: read-only
CREATE POLICY "applicant_profiles_admin_select"
  ON public.applicant_profiles FOR SELECT
  USING (
    EXISTS (
      SELECT 1 FROM public.profiles p
      WHERE p.id = auth.uid() AND p.is_admin = true
    )
  );

-- Service role: unrestricted (for server-side operations)
CREATE POLICY "applicant_profiles_service_role"
  ON public.applicant_profiles FOR ALL
  USING (auth.role() = 'service_role')
  WITH CHECK (auth.role() = 'service_role');

-- updated_at trigger
CREATE TRIGGER set_applicant_profiles_updated_at
  BEFORE UPDATE ON public.applicant_profiles
  FOR EACH ROW EXECUTE FUNCTION public.set_updated_at();

-- Indexes
CREATE INDEX idx_applicant_profiles_user_id ON public.applicant_profiles (user_id);
CREATE INDEX idx_applicant_profiles_state ON public.applicant_profiles (state);
```

  6. `safety_preference:view` — ALLOW if `actor.userId === resource.targetUserId`.
  7. `safety_preference:update` — ALLOW if `actor.userId === resource.targetUserId` only.
  8. `safety_preference:quick_exit` — ALLOW if `actor.userId === resource.targetUserId` only.

  10. `trusted_helper:revoke` — ALLOW if `actor.userId === resource.applicantUserId` OR actor is `platform_admin`.
  11. `trusted_helper:list` — ALLOW if actor has `account_type='applicant'`.
  12. `trusted_helper:act_as` — ALLOW if an active grant exists for `(resource.applicantUserId, actor.userId)` AND `resource.requestedAction` is in the grant's `granted_scope`. The grant lookup is async — the evaluator must be async.
  13. `applicant_bookmark:create` — ALLOW if actor has `account_type='applicant'` AND `actor.userId === resource.applicantUserId`.
  14. `applicant_bookmark:list` — ALLOW if actor has `account_type='applicant'` AND `actor.userId === resource.applicantUserId`.
  15. `applicant_bookmark:delete` — ALLOW if `actor.userId === resource.applicantUserId`.
  16. `applicant_bookmark:reorder` — ALLOW if `actor.userId === resource.applicantUserId`.
- All evaluators return `{ allowed: boolean, reason: string }`. Never throw; always return a result.
- Export a single `evalApplicantDomain` function that dispatches on action string.
